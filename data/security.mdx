## Protections in place

| Layer | Measure |
|---|---|
| **Data in transit** | All traffic is encrypted over TLS/HTTPS |
| **Data at rest** | Database encrypted at rest using AES-256, provided by AWS |
| **Device storage** | Authentication tokens stored in iOS Keychain / Android Keystore via Expo Secure Store |
| **API authentication** | All requests require a signed JSON Web Token (JWT) |
| **Access control** | Collaborative spaces enforce granular role-based permissions |
| **Input validation** | All API requests are validated using Protocol Buffer schema validation (protovalidate) |
| **Internal access** | Access to user data is restricted to authorised personnel on a strict need-to-know basis |

## What we do not do

- We do not store passwords in plaintext — passwords are one-way hashed
- We do not have access to your authentication tokens after they are issued to your device
- We do not use any client-side monitoring that could expose your financial data to third parties

## Limitations

No system is perfectly secure. Transmission over the internet carries inherent risk. If you discover or suspect a security issue, contact us immediately at [support@recontrol.money](mailto:support@recontrol.money).

## Data breach notification

In the event of a breach that poses a risk to users, we will notify affected users without undue delay and comply with applicable notification requirements under the **Information Technology Act, 2000** and the **Digital Personal Data Protection Act, 2023**.

## Storage location

All data is stored in **AWS ap-south-1 (Mumbai, India)**. It does not leave India. There is no secondary copy or offsite backup in another region.
